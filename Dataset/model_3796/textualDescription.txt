The domain model 'Entity' describes the core structural elements and their relationships within the system.

The model defines 2 main class(es).

'Entity_System' represents an important domain concept.
It does not declare any attributes.

'Entity_Entity' represents an important domain concept.
It has the following attributes: 'inDomain', which is of type str; 'name', which is of type str.

Relationships between classes are modeled using associations.
An association connects 'Entity_Entity' and 'Entity_System'. From 'Entity_Entity', the role 'Entity' has cardinality 0..*, and from 'Entity_System', the role 'system' has cardinality 0..*.
An association connects 'Entity_Entity' and 'Entity_System'. From 'Entity_Entity', the role 'entity' has cardinality 0..*, and from 'Entity_System', the role 'System' has cardinality 0..*.
